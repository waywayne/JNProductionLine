# 配置持久化说明

## 概述

产测系统的通用配置已实现自动持久化保存，配置数据不会因应用更新而丢失。

## 配置存储机制

### 使用技术
- **存储方案**: SharedPreferences
- **自动保存**: 每次修改配置后自动保存到本地存储
- **跨平台支持**: Windows、macOS、Linux
- **加载优先级**: 
  1. 优先从注册表/SharedPreferences加载已保存的配置
  2. 如果不存在，则使用代码中定义的默认值
  3. 初始化时会输出日志，显示配置来源

### 存储位置

#### Windows
- 存储在系统注册表中
- 路径: `HKEY_CURRENT_USER\Software\[AppName]\FlutterSharedPreferences`

#### macOS
- 存储在用户偏好设置目录
- 路径: `~/Library/Preferences/[bundle_id].plist`

#### Linux
- 存储在用户数据目录
- 路径: `~/.local/share/[app_name]/shared_preferences.json`

## 使用方法

### 1. 修改配置
1. 点击菜单栏的 **"通用配置"** 按钮
2. 在配置页面修改需要的参数
3. 点击 **"保存配置"** 按钮

### 2. 配置生效
- 配置保存后立即生效
- 新的测试将使用新配置
- 配置会自动持久化到本地存储

### 3. 重置配置
- 在配置页面点击 **"重置为默认值"** 按钮
- 所有配置将恢复为默认值

## 数据安全

### 自动备份
- SharedPreferences 由操作系统管理
- 数据随系统备份自动备份
- 不会因应用卸载重装而丢失（取决于系统备份策略）

### 数据迁移
如需在不同设备间迁移配置：
1. **Windows**: 导出注册表项
2. **macOS**: 复制 plist 文件
3. **Linux**: 复制 shared_preferences.json 文件

## 注意事项

1. **首次使用**: 首次启动时使用默认配置
2. **配置验证**: 保存前会验证输入格式
3. **即时生效**: 配置保存后立即生效，无需重启应用
4. **版本兼容**: 配置在应用更新后保持不变

## 技术实现

### 代码位置
- **配置类**: `lib/config/production_config.dart`
- **配置页面**: `lib/screens/production_config_screen.dart`
- **菜单栏**: `lib/widgets/menu_bar_widget.dart`

### 关键方法
```dart
// 初始化（优先从注册表加载）
await ProductionConfig().init();
// 输出日志：
// ✅ 从注册表/SharedPreferences加载配置
//    硬件版本: 1.0.0
//    产品线: 637
// 或
// ℹ️  未找到保存的配置，使用默认值
//    硬件版本: 1.0.0
//    产品线: 637

// 保存配置（自动持久化到注册表）
await config.setHardwareVersion('1.0.0');
await config.setWuqiPowerThresholdMa(15.0);

// 读取配置（优先从注册表读取，不存在则返回默认值）
String version = config.hardwareVersion;  // 注册表值 ?? 默认值
double threshold = config.wuqiPowerThresholdMa;  // 注册表值 ?? 默认值

// 检查是否已初始化
bool initialized = config.isInitialized;

// 重置配置（将所有值重置为默认值并保存到注册表）
await config.resetToDefaults();
```

### 配置读取逻辑
每个配置项都使用以下模式：
```dart
String get hardwareVersion => 
  _prefs?.getString(_keyHardwareVersion) ?? defaultHardwareVersion;
//  ↑ 从注册表读取                        ↑ 如果不存在，使用默认值
```

这确保了：
1. **优先使用注册表中的值**
2. **注册表不存在时使用默认值**
3. **不会因为注册表为空而崩溃**

## 常见问题

### Q: 配置会丢失吗？
A: 不会。配置使用 SharedPreferences 持久化存储，不会因应用更新而丢失。

### Q: 如何备份配置？
A: 配置随系统备份自动备份。也可以手动复制存储文件。

### Q: 配置文件在哪里？
A: 取决于操作系统，详见"存储位置"章节。

### Q: 如何恢复默认配置？
A: 在通用配置页面点击"重置为默认值"按钮。
